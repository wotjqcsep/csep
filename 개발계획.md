# CSEP 개발 계획

---

## 개발 전략

이 프로젝트는 기능이 매우 많기 때문에 **한 번에 모두 개발하는 것이 아니라 핵심 기능부터 단계적으로 완성**합니다.

각 단계를 완성한 후 실제 업무에 사용하면서 피드백을 받고, 다음 단계로 확장합니다.

---

## 1단계: MVP (핵심 기능)

**목표**: 접수부터 배정, 작업까지 최소 기능으로 실무 사용 가능

### 1단계 포함 기능

✅ **사용자 관리**
- 로그인 (이메일/비밀번호)
- 권한 관리 (대표, 기사, 관리자)
- 기본 프로필

✅ **고객관리 (기본)**
- 고객 추가/수정/조회
- 기본 정보 (이름, 전화, 주소)
- 메모
- 미수금 확인

✅ **전화/SMS 접수**
- 전화 수신 시 번호 인식
- 고객 자동 검색
- 신규 고객 자동 등록
- SMS 자동 분석
- 접수 생성

✅ **컴닥터 접수**
- 특정 번호로 컴닥터 인식
- 문자 자동 분석
- 접수 생성
- 담당 기사 배정

✅ **접수 관리**
- 접수 목록 (상태별 필터)
- 접수 상세 보기
- 상태 변경 (신규 → 배정 → 진행 → 완료)

✅ **기사 배정**
- 기사 목록
- 기사 상태 표시 (대기, 출동, 작업중, 완료, 퇴근)
- 접수를 기사에게 배정
- 기사 스마트폰에 알림

✅ **PC ↔ 스마트폰 동기화**
- 실시간 데이터 동기화
- 오프라인 모드 기본 지원

✅ **스마트폰 앱 (기사용)**
- 접수 목록 보기
- 접수 상세 보기
- 작업 시작/완료 버튼
- 고객 연락처 확인

✅ **PC 프로그램 (대표/관리자용)**
- 대시보드 (오늘 접수, 진행중인 작업, 완료 현황)
- 접수 관리
- 기사 관리
- 고객 기본 조회

### 1단계 기술 스택 (미정, 결정 필요)

| 항목 | 선택지 | 추천 |
|------|--------|------|
| **Backend** | Python/Flask, Node.js/Express, Go/Gin | ? |
| **Frontend (PC)** | 웹 (React/Vue), Electron, 데스크톱 앱 | ? |
| **Mobile** | React Native, Swift/Kotlin, Flutter | ? |
| **Database** | PostgreSQL, MySQL, MongoDB | PostgreSQL 추천 |
| **Server** | Render, Fly.io, Oracle 자체호스팅, AWS | ? |
| **실시간 동기화** | WebSocket, Firebase, Pusher | ? |

### 1단계 예상 기간

**혼자서, 주 40시간 기준:**
- 설계 및 환경 구성: 2주
- Backend API 개발: 4주
- Frontend (PC) 개발: 4주
- Mobile 앱 개발: 4주
- 통합 및 테스트: 2주
- **총 16주 (약 4개월)**

**팀 (3명) 기준:**
- 총 8주 (약 2개월)

## 2단계: 기능 확장

**목표**: 컴퓨터 관리, 이력 관리, 지도 연동

### 2단계 추가 기능

✅ **컴퓨터(장비) 관리**
- 고객별 컴퓨터 등록
- 컴퓨터 상세 정보 (CPU, RAM, SSD, HDD, 메인보드, OS, Office, 백신, 프린터, 네트워크, IP, MAC, 시리얼번호, 조립여부, 판매일, 보증기간)
- 컴퓨터별 이력 조회

✅ **수리/점검 이력**
- 수리 이력 기록
- 점검 이력 기록
- 판매 이력 기록
- 이력 검색 및 필터

✅ **사진 첨부**
- 작업 중 사진 촬영 (모바일)
- 사진 첨부 및 저장
- 사진 갤러리 보기

✅ **지도 연동**
- 고객 주소 클릭 시 지도 열기
- 티맵/카카오맵 길안내
- 기사 위치 공유 및 추적

✅ **기사 위치 추적**
- 기사 위치 실시간 추적
- 지도에 기사 위치 표시
- 고객 주소와의 거리/시간 표시

✅ **통계 (기본)**
- 일일/주간/월간 접수 현황
- 기사별 실적 (작업 수, 매출)
- 고객별 수리 횟수

### 2단계 예상 기간

**1단계 완료 후, 혼자 기준:**
- 약 2~3개월

## 3단계: ERP 통합

**목표**: 판매, 재고, 매출, 세금계산서 등 통합

### 3단계 추가 기능

✅ **판매 관리**
- 컴퓨터 판매 기록
- 부품 판매 기록
- 판매가 설정

✅ **재고 관리**
- 부품 입고/출고 관리
- 재고 현황
- 재고 부족 알림

✅ **매입 관리**
- 공급처 관리
- 부품 매입 기록
- 매입가 관리

✅ **거래처 관리**
- 거래처 정보 등록
- 거래처별 거래 이력

✅ **견적서 자동 생성**
- 고객/부품 선택 시 자동 견적서 생성
- PDF 다운로드

✅ **거래명세서 자동 생성**
- 거래 항목별 자동 생성
- PDF 다운로드

✅ **세금계산서 연동** (선택)
- 세무서 자동 신고 (API 연동 필요)
- 또는 수동으로 생성

✅ **결제 기능**
- 현금, 계좌이체, 카드결제, 간편결제, 외상 기록
- 미수금 추적

✅ **매출 통계**
- 월별/분기별 매출
- 기사별 매출
- 고객별 매출

✅ **자동 영수증/확인서 생성**
- 작업 완료 후 자동 생성
- PDF 저장

### 3단계 예상 기간

**2단계 완료 후, 혼자 기준:**
- 약 2~3개월

## 우선순위 기준

### 1단계에서 꼭 필요한 것
1. **접수 시스템** (전화, SMS, 컴닥터)
2. **기사 배정** (기사에게 일 배치)
3. **PC ↔ 모바일 동기화** (리얼타임)
4. **기본 고객관리** (고객 정보)

### 1단계에서 생략 가능한 것
- 컴퓨터 상세 관리 → 2단계로
- 지도 연동 → 2단계로
- ERP 기능 → 3단계로
- AI 기능 → 나중

- 한 명이 관리 가능해야 함
- 문서화 중요
